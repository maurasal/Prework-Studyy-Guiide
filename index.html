HTML

  • The head element contains information about the webpage.
  • Only use h1 for the main header and you'll usually only use it once.
  • An element is from opening tag to closing tag.
  • A tag is what appears only inside the angle brackets i.e. "li" or "h2".
  • Elements are referred to using parent/child relationships.
  • The body element represents the visible content shown to the user.

CSS

  • A mergin idicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside of the element.
  • External style sheet - Keeps all CSS on a seperate file from the HTML. This allows for easier editing.
  • Commands in CSS are created with a sector and a declaration. The sector defines the element or attributes to which the declarations or rules will apply.

Git

  • git status: checks what branch we are currently working on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git commit -m "description": commit your content.
  • A repository is a folder or system that contains files of code.
  • Markdown .md is a file where you write and edit the code to be sent or seen by others. Can easily be converted to code.
  • A pull request asks changes on one branch to merge with another branch.
  • Branches allow you to work on and edit parts of your code without it affecting the main code, which might be live to users.
